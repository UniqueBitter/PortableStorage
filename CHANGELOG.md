# 更新日志

本文件从此版本开始记录。版本号 `1.0.N` 的 N 是构建计数(每次构建 +1),下面按日期归档改动。

格式:**新增** = 新功能 ・ **优化** = 改进/性能 ・ **修复** = Bug ・ **构建** = 工程/打包。

---

## 2026-07-04

### 新增
- **F 键一键全部收纳**:不用开界面,任何地方按 `F` 把背包物品全部收纳(锁定格不收、已归类各回原标签、新种类进未分类)。可在按键设置改键。
- **标签换序**:中键点自定义标签 → 输入数字设置新序号。
- **Shift+左键批量取出**:在仓库里按住 Shift+左键划过,连续取出到背包(取出的格先空着,松手后按当前排序重排)。
- **CustomNPCs 软依赖**:接入 CNPC API(`compileOnly` 瘦身 jar),用 `CnpcCompat.LOADED` 保护,没装 CNPC 的地图也能正常加载。

### 优化
- **全部收纳不再"穿透"分类**:批量存入时,已归到某标签的物品**保留原标签**,只有新/未分类的才进未分类(不再一股脑塞进当前正看的标签)。
- **「匹配已有物品」模式**(原「泰拉模式」重命名):只补充仓库已有种类的物品;实现改为**一次查库 + 批量写入**,大背包更跟手。
- **UI 主题(main 分支)**:仓库背景换成整合包的 generic_54 木质箱子皮肤,按钮/标签/滚动条改木色,文字纯白。
- 容量行文案:无限模式显示「还可存 无限」,容量制显示「还可存 X 种」。

### 修复
- **标签页内排序失效**:自定义标签之前被强制按插入顺序排、忽略排序设置;现在所有视图都听你选的排序(想要插入顺序就选「时间」)。
- **分类被穿透**:见上「全部收纳」优化(重启后一键收纳把币等已归类物品冲成当前标签的问题)。
- 容量显示「还可存 0 种」:无限模式哨兵值未特判导致,现修正为「无限」。
- 标签文字发灰:`GuiContainer` 渲染物品后残留 `GL_LIGHTING` 把白字压暗,画标签前先关闭光照。
- 物品 tooltip 被右侧标签列遮住:移到最上层绘制。

### 构建
- **版本号自增**:改用 `version.txt` 计数,产物固定为 `1.0.N` 且每次构建 +1(不再是 git 哈希串)。
- **自动装包**:`build`/`assemble` 完成后自动删除整合包 `mods/` 里旧的 PortableStorage jar 并装入新构建(游戏开着锁住时安全跳过)。
- **general-backpack 分支**:同步 main 的全部功能,保留原灰色 UI(不带木质背景/按钮配色)。
